Faculty Progress Report

Staff upload faculty CVs and FAR PDFs; the tool extracts and counts publications, grants, students, and service entries using deterministic regex (with an optional Claude classifier for semantic rules) and outputs a consolidated Excel workbook. A PDF quality gate with Tesseract OCR fallback handles scanned and low-quality pages — every count traces back to exact matched entries.

EIRA

RAG-powered chatbot for the ECE department's public website — answers questions about faculty, research areas, courses, and lab opportunities using a fine-tuned TAMU ECE embedding model, graph-based retrieval with area aliases and intersection query support, and the TAMU AI LLM. The index rebuilds nightly at 2 AM automatically.

EIRA is self-healing: users can flag a wrong answer directly in the chat UI, which triggers an automated Codex agent at the backend. The agent diagnoses the root cause — missing faculty entry, wrong area tag, retrieval gap — and deploys a fix without any human intervention.

GradAPPTriage

Automates the first-pass review of graduate applications. Ingests CAS application packets (PDF) alongside a structured applicant CSV — merging by cas_id — then uses a content-based classifier to automatically separate resumes, statements of purpose, and letters of recommendation even without clear headings. Outputs structured candidate profiles with clean motivation quotes, GPA breakdowns, and flagged anomalies (e.g., invalid GRE totals) ready for the admissions committee.
